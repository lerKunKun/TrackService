# 数据库管理工具字符集配置指南

## 问题说明
如果您的数据库管理工具显示中文注释为乱码，请按照以下步骤配置字符集。

## 各类工具配置方法

### 1. Navicat / Navicat Premium
1. 右键点击数据库连接 → **编辑连接**
2. 在 **高级** 选项卡中
3. 找到 **使用 MySQL 字符集编码** 选项
4. 设置为：`utf8mb4` 或 `Auto`
5. 点击 **测试连接** 确认，然后保存

### 2. MySQL Workbench
1. 编辑连接 → **Advanced** 标签页
2. 在 **Others** 部分添加：
   ```
   OPT_CHARSET_NAME=utf8mb4
   ```
3. 或在 **Connection** 标签页的 **Default Character Set** 选择 `utf8mb4`

### 3. DBeaver
1. 编辑连接 → **Driver properties** 标签页
2. 添加或修改以下属性：
   - `characterEncoding` = `UTF-8`
   - `useUnicode` = `true`
3. 点击 **Test Connection** 测试

### 4. HeidiSQL
1. 会话管理器 → 选择连接 → **设置**
2. 在 **高级** 选项卡
3. **服务器字符集** 选择 `utf8mb4`
4. **客户端字符集** 也选择 `utf8mb4`

### 5. phpMyAdmin
在 `config.inc.php` 文件中添加：
```php
$cfg['DefaultCharset'] = 'utf8mb4';
$cfg['DefaultConnectionCollation'] = 'utf8mb4_unicode_ci';
```

### 6. MySQL 命令行
每次连接时使用：
```bash
docker exec mysql mysql -uroot -p123456 \
  --default-character-set=utf8mb4 \
  logistics_system
```

或者在命令行连接后执行：
```sql
SET NAMES utf8mb4;
```

### 7. JDBC 连接字符串（Java应用）
```java
jdbc:mysql://localhost:3306/logistics_system?characterEncoding=UTF-8&useUnicode=true
```

### 8. Python (mysql-connector, PyMySQL)
```python
connection = mysql.connector.connect(
    host='localhost',
    user='root',
    password='123456',
    database='logistics_system',
    charset='utf8mb4'
)
```

## 验证方法

配置完成后，在工具中执行以下SQL验证：

```sql
SHOW TABLE STATUS FROM logistics_system 
WHERE Name IN ('liquid_schema_cache', 'theme_migration_rules');
```

应该正确显示：
- `liquid_schema_cache` - **Liquid Schema缓存表**
- `theme_migration_rules` - **主题迁移规则表**

## 如果仍显示乱码

1. **退出并重新连接**数据库
2. **刷新**数据库连接或重启管理工具
3. 检查工具本身的**界面显示编码**设置（通常在首选项中）
4. 确保操作系统的**区域和语言设置**支持中文
